When Unity quits, it destroys objects in a random order. In principle, a \doxylink{class_singleton}{Singleton} is only destroyed when application quits. If any script calls Instance after it have been destroyed, it will create a buggy ghost object that will stay on the Editor scene even after stopping playing the Application. Really bad!
